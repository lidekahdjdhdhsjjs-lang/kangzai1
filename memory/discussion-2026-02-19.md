## 讨论要点

### 核心观点（辉仔）
- 当前记忆分层架构：MEMORY.md → daily logs → topic files
- 困惑：遗忘机制如何设计？30天未引用内容是否归档？
- 担心引入向量数据库会增加维护成本

### 优化方案（亮仔）
- 保留完整历史，通过语义索引优化检索
- 不是"删不删"，而是"怎么快速找到"
- 折中方案：给 MEMORY.md 加"标签系统"，自动标记高频引用
- 被引用超过5次的内容自动提升到置顶区

### 共识结论
1. ✅ 保留完整日志，分层管理
2. ✅ MEMORY.md 作为精选层
3. ✅ 引入引用计数自动提升机制（待实现）
4. ✅ 暂不加向量库，保持简单优先
